# 003: .temp/ — временные файлы агента внутри проекта

## Проблема

Сейчас агент использует системную временную директорию (`/tmp`, `%TEMP%`)
для временных файлов. Это приводит к:

- Срабатыванию эвристик антивирусов (активность в temp)
- Потере временных артефактов между сессиями (очистка temp)
- Разбросанности: temp файлы не привязаны к проекту
- Невозможность проверить что делал агент (temp очищен)

## Идея

Создать `.temp/` в корне проекта для всех временных файлов агента.
Добавить `.temp/` в `.gitignore` проекта.

- `.temp/` — не внутри `.agent/`, а в корне (на одном уровне с `.agent/`)
- Не коммитится (gitignore)
- Может быть очищен пользователем без потери артефактов сессии
- Доступен из любой точки проекта

## Механизм

### При install.sh / INIT

1. Создать `.temp/` в корне проекта (если нет)
2. Создать/дополнить `.gitignore` строкой `.temp/` (если нет)

### Использование

Агент пишет временные файлы в `.temp/` вместо `/tmp` или `%TEMP%`:

```
.temp/
  downloads/            # временно скачанные файлы
  patches/              # временные патчи и diff
  cache/                # кеш анализа, промежуточные результаты
  agent-session-xxx/    # временные файлы конкретной сессии
```

### Интеграция

- В `BOUNDARIES.md` — разрешить создание `.temp/` и запись `.gitignore`
- В `META_AGENT_GUIDE.md` — упомянуть `.temp/` в INIT: создать + gitignore
- В `install.sh` / `install.ps1` — создавать `.temp/`, добавлять в `.gitignore`
- Агент во всех фазах использует `.temp/` вместо системного temp

## Связь с другими фичами

Независимая фича. Не требует 001 или 002.

## Открытые вопросы

- Нужна ли авто-очистка `.temp/` старше N дней?
- Может ли orchestrator очищать `.temp/` между сессиями?
- Нужно ли логировать что было в `.temp/` для аудита?
